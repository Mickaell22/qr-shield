## Hoja: 1.1

|  |  | Análisis del Problema Central |  |  |  |  |
| --- | --- | --- | --- | --- | --- | --- |
|  |  |  | Proyecto: Implementación de una solución multiplataforma para la identificación de códigos QR y detección temprana de amenazas asociadas a ataques de quishing |  |  |  |
|  |  | 1. | Descripción del problema central |  |  |  |
|  |  | 1.1 | Análisis Causal |  |  |  |
|  |  |  |  | N° | Causas | Consecuencias |
|  |  |  |  | 1 | El contenido de un código QR permanece oculto hasta ser escaneado | El usuario accede al destino sin poder evaluar el riesgo previamente |
|  |  |  |  | 2 | Confianza ciega del usuario al escanear códigos QR | Mayor probabilidad de caer en sitios fraudulentos por desconocimiento |
|  |  |  |  | 3 | Los filtros de seguridad tradicionales interpretan el QR como una simple imagen | Los ataques de quishing evaden las defensas convencionales de correo y navegación |
|  |  |  |  | 4 | Facilidad para alterar o suplantar un código QR físico (adhesivos sobre el original) | El usuario es redirigido a destinos maliciosos sin advertirlo |
|  |  |  |  | 5 | Ausencia de una capa intermedia entre el escaneo y el acceso al destino | No existe ninguna barrera de protección antes de visitar el sitio |
|  |  |  |  | 6 | Inexistencia de herramientas locales accesibles para verificar un código QR | El usuario queda sin alternativas de protección integradas a su uso cotidiano |
|  |  |  |  | 7 | Alta exposición a códigos QR en espacios físicos abiertos del campus | Aumento de la superficie de ataque dentro de la comunidad universitaria |
|  |  |  |  | 8 | Perfeccionamiento continuo de las técnicas de ingeniería social | Ataques de quishing cada vez más difíciles de identificar por el usuario |
|  |  |  |  | 9 | Desconocimiento general sobre la existencia del quishing como amenaza | La víctima no toma precauciones por ignorar el riesgo |
|  |  |  |  | 10 | Uso de URL acortadas o enmascaradas dentro de los códigos QR | Dificultad para reconocer el destino real antes de acceder a él |
