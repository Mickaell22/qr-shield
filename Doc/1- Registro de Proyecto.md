|  | Problemática | El uso cotidiano de códigos QR se ha consolidado en Ecuador mediante soluciones como Deuna (Banco Pichincha), Payphone y la banca móvil, especialmente entre jóvenes de 18 a 35 años, que representan cerca del 67 % de los usuarios de billeteras digitales. Esta masificación ha sido aprovechada por ciberdelincuentes a través del quishing, una variante de phishing que sustituye o falsifica códigos QR para redirigir al usuario a sitios fraudulentos o instalar software malicioso en el dispositivo. Entre agosto y noviembre de 2025 el volumen mundial de ataques de quishing se multiplicó por cinco, y los filtros de seguridad tradicionales no detectan esta amenaza, ya que el código QR es tratado como una imagen inofensiva cuyo contenido no se analiza antes de ser escaneado.    En Ecuador, el Ministerio de Telecomunicaciones alertó en julio de 2025 sobre el crecimiento del quishing en comercios y espacios públicos con códigos QR adulterados, y entidades como Banco Pichincha han publicado material educativo sobre fraudes basados en QR. Pese a este reconocimiento institucional, no existen herramientas accesibles en el mercado local que permitan al usuario verificar en tiempo real la seguridad de un código QR antes de interactuar con él. Los estudiantes de la Facultad de Ciencias Matemáticas y Físicas de la Universidad de Guayaquil constituyen un escenario particularmente expuesto a esta amenaza, ya que interactúan diariamente con códigos QR para acceso a la red wifi institucional, consumo en cafeterías y comercios cercanos al campus, trámites académicos, carteleras informativas y banca móvil, en un entorno físico abierto donde la suplantación o adulteración de un código impreso resulta trivial. Esta concentración de usuarios con alta exposición y limitadas herramientas de verificación justifica el desarrollo de un mecanismo confiable y de uso cotidiano que permita a esta población estudiantil detectar códigos QR maliciosos antes de ser víctimas de un ataque de quishing. | 0 |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  | 0.9 |

|  | Objetivos | General: 1. Desarrollar una solución multiplataforma para la identificación de códigos QR y la detección temprana de amenazas asociadas a ataques de quishing, dirigida a los estudiantes de la Facultad de Ciencias Matemáticas y Físicas de la Universidad de Guayaquil.  Específicos: 1. Analizar las técnicas actuales de detección de códigos QR maliciosos y los principales vectores de ataque de quishing, mediante una revisión de literatura científica y técnica, para establecer los criterios de clasificación que usará el sistema. 2. Diseñar la arquitectura del sistema y el motor de detección de URL maliciosas, integrando servicios externos de inteligencia de amenazas con heurísticas propias, y definiendo las interfaces de la extensión para Chromium y la aplicación móvil desarrollada en Flutter. 3. Implementar la extensión para navegadores basados en Chromium y la aplicación móvil Android, desarrollando el motor de detección compartido y los módulos de escaneo, análisis y alerta al usuario en tiempo real. 4. Validar técnicamente el funcionamiento del sistema mediante pruebas con un conjunto representativo de códigos QR maliciosos y legítimos en escenarios de uso de los estudiantes de la Facultad de Ciencias Matemáticas y Físicas de la Universidad de Guayaquil, evaluando métricas de precisión, exhaustividad y tiempo de respuesta. |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |
|  | Justificación e importancia | El desarrollo de esta solución tecnológica se justifica por el crecimiento acelerado del quishing como vector de ataque en Ecuador, reconocido oficialmente por el Ministerio de Telecomunicaciones, y por la ausencia de herramientas locales que permitan a los usuarios verificar la seguridad de un código QR antes de ser escaneado. La experiencia cotidiana confirma la necesidad: incluso usuarios con formación técnica recurren a prácticas manuales como tomar capturas de pantalla del código para analizar la URL destino, lo que evidencia que el problema no se resuelve con educación del usuario sino con herramientas integradas al flujo natural de uso. El aporte del proyecto radica en ofrecer una solución de doble cobertura (extensión para navegadores basados en Chromium y aplicación móvil Android) articulada sobre un motor de detección que combina servicios externos de inteligencia de amenazas con heurísticas propias, brindando una capa de protección tanto en el escritorio como en el dispositivo móvil. La pertinencia institucional del proyecto se refuerza al centrar la implementación y validación en los estudiantes de la Facultad de Ciencias Matemáticas y Físicas de la Universidad de Guayaquil, un entorno de alta densidad de usuarios y exposición continua a códigos QR en escenarios académicos, administrativos y comerciales del campus, donde una herramienta accesible de verificación aporta valor directo de ciberseguridad a los estudiantes de la facultad. La viabilidad del proyecto se sostiene en la formación del autor en Ingeniería de Software y en ciberseguridad ofensiva (certificación CEH), respaldada por un laboratorio personal de pruebas, lo que asegura la capacidad técnica para integrar correctamente APIs de detección, diseñar heurísticas efectivas y validar el sistema frente a códigos QR maliciosos reales. |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |  |
